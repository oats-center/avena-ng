# Avena ns-3 Driver

This directory contains a real ns-3 C++ driver implementation for `avena-testbed`.

The driver:

- parses the JSON runtime config generated by `avena-testbed`
- creates one ns-3 node per testbed node
- installs mobility from per-node trace files or fixed positions
- creates WiFi networks for `links` and `bridges`
- attaches each radio to Linux TAP using `TapBridge` (`UseLocal` mode)
- runs with `RealtimeSimulatorImpl`
- emits newline-delimited JSON telemetry events on stdout

## Files

- `avena-ns3-driver.cc`: driver source code
- `avena-ns3-driver.sh`: wrapper script to run the driver from an ns-3 tree

## Build and Run (ns-3 tree)

From your ns-3 checkout (`$NS3_ROOT`):

1. Copy the source into `scratch/`:

```bash
cp /path/to/avena/ns3-driver/avena-ns3-driver.cc "$NS3_ROOT"/scratch/
```

2. Build ns-3 scratch programs:

```bash
cd "$NS3_ROOT"
./ns3 build
```

3. Make wrapper executable and point testbed to it:

```bash
chmod +x /path/to/avena/ns3-driver/avena-ns3-driver.sh
export AVENA_NS3_ROOT="$NS3_ROOT"
export AVENA_NS3_DRIVER_BIN="/path/to/avena/ns3-driver/avena-ns3-driver.sh"
```

Then run `avena-testbed` in ns3 mode. The existing Rust runner will pass `--config` and `--duration-secs` to the wrapper.

## Dependencies

- ns-3 with modules used by the driver (`core`, `network`, `mobility`, `wifi`, `tap-bridge`, `spectrum`)
- C++17 compiler
- `nlohmann/json.hpp` available in include path

On Debian/Ubuntu, install the JSON header via:

```bash
sudo apt-get install -y nlohmann-json3-dev
```

## Event Shape

Driver stdout emits JSON lines such as:

- realtime lag: `{"type":"realtime","sim_ms":...,"wall_ms":...,"lag_ms":...}`
- mobility snapshots: `{"type":"mobility","node":"nodeA","x_m":...}`
- L2 readiness: `{"type":"l2","event":"l2_ready","node":"nodeA","radio":"wifi0"}`

The Rust testbed republishes these events to NATS under `avena.v1.<run_id>.ns3.*`.
